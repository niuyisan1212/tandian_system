## 1. 项目概述

### 1.1 项目背景
探店路线规划系统是一个帮助用户高效规划探店行程的工具。用户可以管理待探店铺信息，创建探店清单，系统自动规划最优路线并推荐公共交通方式。

### 1.2 技术栈确认

**后端技术栈**
- 框架：Spring Boot 3.x
- 数据库：MySQL 8.0
- ORM：MyBatis-Plus
- 构建工具：Maven
- 日志框架：SLF4J + Logback

**前端技术栈**
- 框架：Vue 3 + Vite
- UI组件库：Ant Design Vue 4.x
- 地图SDK：高德地图API（首版使用模拟数据，预留接口）
- 状态管理：Pinia
- 路由：Vue Router 4

## 2. 功能需求

### 2.1 核心功能模块

#### 模块一：店铺管理
- **SHOP-01 添加店铺**：表单录入店铺信息，包含名称、地址、联系方式、营业时间、备注、过期时间、探店状态
- **SHOP-02 编辑店铺**：支持修改所有字段
- **SHOP-03 删除店铺**：永久删除，若被清单引用需提示
- **SHOP-04 店铺列表**：表格展示，支持筛选和搜索
- **SHOP-05 批量导入导出**：Excel导入导出功能

**关键业务规则**：
- 有效待探店店铺 = 未探店状态 AND (未设置过期时间 OR 过期时间>当前时间)
- 首版使用模拟的经纬度数据（随机生成或预设），预留地理编码接口

#### 模块二：探店清单管理
- **LIST-01 创建清单**：指定清单名称、起点、选择待探店铺
- **LIST-02 清单列表**：展示所有清单，支持状态筛选
- **LIST-03 清单详情**：查看起点、店铺列表、路线方案
- **LIST-04 完成清单**：标记为已完成/已取消，自动更新店铺状态
- **LIST-05 编辑清单**：修改清单内容
- **LIST-06 删除清单**：删除清单（不影响店铺状态）

**状态同步规则**：
- 清单标记"已完成"时，其所有店铺自动变为"已探店"
- 其他清单若包含这些店铺，需提示重新规划

#### 模块三：路线规划
- **ROUTE-01 自动规划**：创建清单时自动生成路线方案
- **ROUTE-02 多方案生成**：提供2-3种路线方案
  - 时间优先方案
  - 成本优先方案  
  - 均衡方案
- **ROUTE-03 路线展示**：显示总距离、总耗时、总花费、访问顺序
- **ROUTE-04 分段指引**：每段行程的交通方式、详细路线、耗时

**首版实现策略**：
- 使用模拟数据生成路线规划结果
- 预留外部API接口，后期可对接高德地图API
- 基于预设算法计算距离和路线

#### 模块四：地图可视化
- **MAP-01 首页地图**：展示所有待探店位置
- **MAP-02 清单联动**：选择清单后高亮该清单店铺
- **MAP-03 路线展示**：在地图上绘制规划路线
- **MAP-04 店铺标注**：显示起点和店铺位置，标注访问顺序

**首版实现策略**：
- 使用高德地图JS API进行地图展示（免费配额足够）
- 路线规划使用模拟数据
- 预留后端API接口配置

#### 模块五：历史记录
- **HIST-01 已完成清单**：查看历史探店记录
- **HIST-02 基础统计**：累计探店数、总里程、总花费

## 3. 数据库设计

### 3.1 核心表结构

**店铺表 (shop)**
```sql
- id: BIGINT 主键
- name: VARCHAR(100) 店铺名称
- address: VARCHAR(255) 地址
- longitude: DECIMAL(10,7) 经度
- latitude: DECIMAL(10,7) 纬度
- geo_source: VARCHAR(20) 坐标来源
- phone: VARCHAR(20) 联系电话
- business_hours: VARCHAR(100) 营业时间
- remark: TEXT 备注
- visit_status: TINYINT 探店状态(0未探店/1已探店)
- expire_time: DATETIME 过期时间
- created_at: DATETIME 创建时间
- updated_at: DATETIME 更新时间
```

**探店清单表 (shopping_list)**
```sql
- id: BIGINT 主键
- name: VARCHAR(100) 清单名称
- task_date: DATE 任务日期
- start_lng: DECIMAL(10,7) 起点经度
- start_lat: DECIMAL(10,7) 起点纬度
- start_address: VARCHAR(255) 起点地址
- status: TINYINT 状态(0待执行/1已完成/2已取消)
- selected_plan_type: VARCHAR(20) 选择的方案类型
- created_at: DATETIME 创建时间
- updated_at: DATETIME 更新时间
```

**清单-店铺关联表 (list_shop)**
```sql
- id: BIGINT 主键
- list_id: BIGINT 清单ID
- shop_id: BIGINT 店铺ID
- visit_order: INT 访问顺序
- is_skipped: BOOLEAN 是否跳过
```

**路线方案表 (route_plan)**
```sql
- id: BIGINT 主键
- list_id: BIGINT 清单ID
- plan_type: VARCHAR(20) 方案类型
- total_distance_m: INT 总距离(米)
- total_duration_sec: INT 总耗时(秒)
- total_cost: DECIMAL(10,2) 总花费
- route_json: JSON 路线详情
- created_at: DATETIME 创建时间
```

### 3.2 数据库特点
- 符合第三范式
- 使用适当的索引优化查询性能
- 预留扩展字段

## 4. 接口设计

### 4.1 RESTful API规范

**店铺相关接口**
- GET /api/shops - 获取店铺列表
- GET /api/shops/{id} - 获取店铺详情
- POST /api/shops - 创建店铺
- PUT /api/shops/{id} - 更新店铺
- DELETE /api/shops/{id} - 删除店铺
- POST /api/shops/import - 批量导入
- GET /api/shops/export - 批量导出

**清单相关接口**
- GET /api/lists - 获取清单列表
- GET /api/lists/{id} - 获取清单详情
- POST /api/lists - 创建清单
- PUT /api/lists/{id} - 更新清单
- DELETE /api/lists/{id} - 删除清单
- PUT /api/lists/{id}/complete - 标记完成
- PUT /api/lists/{id}/cancel - 标记取消

**路线规划接口**
- POST /api/routes/plan - 规划路线
- GET /api/routes/{listId} - 获取清单的路线方案

**统计接口**
- GET /api/statistics/overview - 获取统计数据
