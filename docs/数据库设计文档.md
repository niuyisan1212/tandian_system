# 探店路线规划系统 - 数据库设计文档

> **版本**：V1.0  
> **日期**：2026-06-08  
> **数据库**：MySQL 8.0

---

## 1. 数据库概述

### 1.1 数据库名称
`tandian_system`

### 1.2 字符集
- 字符集：utf8mb4
- 排序规则：utf8mb4_unicode_ci

### 1.3 设计原则
- 符合数据库第三范式
- 表名使用小写字母+下划线命名
- 字段名使用小写字母+下划线命名
- 主键统一使用自增BIGINT类型
- 时间字段统一使用DATETIME类型
- 适当的索引优化查询性能

---

## 2. 表结构设计

### 2.1 店铺表 (shop)

**表说明**：存储店铺基本信息、位置坐标、探店状态等

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | BIGINT | - | 否 | 自增 | 主键ID |
| name | VARCHAR | 100 | 否 | - | 店铺名称 |
| address | VARCHAR | 255 | 否 | - | 店铺地址 |
| longitude | DECIMAL | (10,7) | 否 | - | 经度（GCJ-02坐标系） |
| latitude | DECIMAL | (10,7) | 否 | - | 纬度（GCJ-02坐标系） |
| geo_source | VARCHAR | 20 | 是 | 'manual' | 坐标来源（geocode/manual/accurate） |
| phone | VARCHAR | 20 | 是 | NULL | 联系电话 |
| business_hours | VARCHAR | 100 | 是 | NULL | 营业时间 |
| remark | TEXT | - | 是 | NULL | 备注信息 |
| visit_status | TINYINT | - | 否 | 0 | 探店状态（0未探店/1已探店） |
| expire_time | DATETIME | - | 是 | NULL | 过期时间（NULL表示永不过期） |
| created_at | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | 否 | CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- INDEX idx_visit_status (visit_status)
- INDEX idx_expire_time (expire_time)
- INDEX idx_created_at (created_at)

**业务规则**：
- 有效待探店店铺：visit_status = 0 AND (expire_time IS NULL OR expire_time > NOW())
- 经纬度使用GCJ-02坐标系（国测局坐标系）
- geo_source记录坐标来源，便于后续优化

---

### 2.2 探店清单表 (shopping_list)

**表说明**：存储探店清单信息，包含起点、状态等

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | BIGINT | - | 否 | 自增 | 主键ID |
| name | VARCHAR | 100 | 否 | - | 清单名称 |
| task_date | DATE | - | 否 | - | 任务日期 |
| start_lng | DECIMAL | (10,7) | 是 | NULL | 起点经度 |
| start_lat | DECIMAL | (10,7) | 是 | NULL | 起点纬度 |
| start_address | VARCHAR | 255 | 是 | NULL | 起点地址文本 |
| status | TINYINT | - | 否 | 0 | 状态（0待执行/1已完成/2已取消） |
| selected_plan_type | VARCHAR | 20 | 是 | NULL | 选择的方案类型 |
| created_at | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | 否 | CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- INDEX idx_task_date (task_date)
- INDEX idx_status (status)
- INDEX idx_created_at (created_at)

**业务规则**：
- 状态转换：待执行 → 已完成/已取消
- 标记完成时自动更新关联店铺的探店状态

---

### 2.3 清单-店铺关联表 (list_shop)

**表说明**：存储清单与店铺的多对多关系，包含访问顺序和跳过标记

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | BIGINT | - | 否 | 自增 | 主键ID |
| list_id | BIGINT | - | 否 | - | 清单ID |
| shop_id | BIGINT | - | 否 | - | 店铺ID |
| visit_order | INT | - | 是 | NULL | 访问顺序（1开始） |
| is_skipped | TINYINT | - | 否 | 0 | 是否跳过（0否/1是） |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_list_shop (list_id, shop_id)
- INDEX idx_list_id (list_id)
- INDEX idx_shop_id (shop_id)

**外键约束**：
- FOREIGN KEY (list_id) REFERENCES shopping_list(id) ON DELETE CASCADE
- FOREIGN KEY (shop_id) REFERENCES shop(id) ON DELETE CASCADE

**业务规则**：
- 一个店铺可以在多个清单中
- 一个清单可以包含多个店铺
- visit_order记录在当前路线方案中的访问顺序
- is_skipped标记用户临时排除的店铺

---

### 2.4 路线方案表 (route_plan)

**表说明**：存储清单的路线规划方案详情

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | BIGINT | - | 否 | 自增 | 主键ID |
| list_id | BIGINT | - | 否 | - | 清单ID |
| plan_type | VARCHAR | 20 | 否 | - | 方案类型（time_first/cost_first/balanced） |
| total_distance_m | INT | - | 否 | 0 | 总距离（米） |
| total_duration_sec | INT | - | 否 | 0 | 总耗时（秒） |
| total_cost | DECIMAL | (10,2) | 否 | 0.00 | 总花费（元） |
| route_json | JSON | - | 是 | NULL | 路线详情JSON |
| created_at | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- PRIMARY KEY (id)
- INDEX idx_list_id (list_id)
- UNIQUE KEY uk_list_plan_type (list_id, plan_type)

**外键约束**：
- FOREIGN KEY (list_id) REFERENCES shopping_list(id) ON DELETE CASCADE

**route_json 结构**：
```json
{
  "sequence": [101, 102, 103],
  "segments": [
    {
      "from_type": "start",
      "to_shop_id": 101,
      "mode": "subway",
      "instruction": "步行300米至人民广场站，乘1号线2站至黄陂南路站",
      "duration_sec": 900,
      "distance_m": 2500,
      "cost": 3.00,
      "steps": []
    }
  ]
}
```

## 4. 数据完整性约束

### 4.1 外键约束
- list_shop.list_id → shopping_list.id (CASCADE DELETE)
- list_shop.shop_id → shop.id (CASCADE DELETE)
- route_plan.list_id → shopping_list.id (CASCADE DELETE)

### 4.2 唯一约束
- list_shop(list_id, shop_id)：同一清单中同一店铺只能出现一次
- route_plan(list_id, plan_type)：同一清单的同类型方案只能有一个

### 4.3 检查约束
- shop.visit_status IN (0, 1)
- shopping_list.status IN (0, 1, 2)
- route_plan.plan_type IN ('time_first', 'cost_first', 'balanced')

## 5. 索引策略

### 5.1 主键索引
所有表都有自增主键索引

### 5.2 业务查询索引
- shop表：visit_status, expire_time, created_at
- shopping_list表：task_date, status, created_at
- list_shop表：list_id, shop_id
- route_plan表：list_id, (list_id, plan_type)

### 5.3 性能优化建议
- 查询有效待探店店铺时使用复合索引 (visit_status, expire_time)
- 查询清单列表时使用索引 (status, task_date, created_at)
