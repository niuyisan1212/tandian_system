## 🎯 功能实现

### 1. 地理编码
**触发场景**：创建/编辑店铺时输入地址

**流程**：
1. 用户输入地址文本
2. 后端调用高德地图地理编码API
3. 解析为经纬度坐标
4. 存储到数据库

**降级策略**：API失败时使用默认坐标

### 2. 路线规划
**触发场景**：创建清单时自动规划

**流程**：
1. 获取清单中的店铺列表
2. 使用TSP算法优化访问顺序
3. 对每段行程调用高德地图API
4. 根据距离选择交通方式
5. 生成3种路线方案

**交通方式选择**：
- ≤ 800米：步行
- 800-3000米：步行（非成本优先）
- > 3000米：公交/地铁

**路线方案**：
- 时间优先：地铁/快速公交
- 成本优先：普通公交
- 均衡方案：综合考量

### 3. 地图展示
**首页地图**：
- 展示所有有效待探店店铺
- 点击标记查看店铺信息
- 选择清单后高亮显示

**详情页地图**（待完善）：
- 展示清单店铺位置
- 绘制规划路线
- 显示交通方式

## 📊 技术实现

### 后端技术栈
- Spring Boot 2.7.18
- RestTemplate（HTTP客户端）
- FastJSON 2.0（JSON处理）
- 高德地图 Web服务API

### 前端技术栈
- Vue 3.4
- 高德地图 JS API 2.0
- Ant Design Vue 4.0

### API调用
**地理编码API**：
```
GET https://restapi.amap.com/v3/geocode/geo?key={key}&address={address}&city={city}
```

**步行路径规划API**：
```
GET https://restapi.amap.com/v3/direction/walking?key={key}&origin={lng,lat}&destination={lng,lat}
```

**公交路径规划API**：
```
GET https://restapi.amap.com/v3/direction/transit/integrated?key={key}&origin={lng,lat}&destination={lng,lat}&city={city}&strategy={strategy}
```
